PART 01 · 安全评估
01
架构层面 · Architecture

沙箱环境隔离，不对外网暴露

  • 每个 OpenClaw 实例独立运行，容器间网络隔离，多租户隔离
  • 默认不出外网，通过 AI 安全护栏转接模型
  • 本地部署，资源可集中化进行安全管控
对应公司安全需求

部署在内网，可以和必要的系统/数据打通，无需外网暴露数据接口。

缓释的原生 OpenClaw 风险
  • 模型安全可使用非公司提供模型，带来潜在模型安全风险
  • 系统安全公网暴露与远程入侵风险
  • 法律合规责任边界风险
02
配置管控 · Configuration

安全加固镜像，集中化管理

  • 集成漏洞扫描、密钥扫描、环境分离、防密钥泄露等安全配置
  • 非 root 低权限运行、只读根文件系统、WorkspaceOnly 目录限制，白名单控制
对应公司安全要求

安全基线配置、环境资源配置、网络控制集中收敛到 DT 管控

缓释的原生 OpenClaw 风险
  • 数据安全企业敏感数据、凭证与密钥明文泄露风险
  • 智能体安全权限过高风险
  • 智能体安全安全机制绕过风险
  • 系统安全安全漏洞风险
03
供应链安全 · Supply Chain

内部 Skill Hub，所有 Skills 需安全审查

  • Skills 评级过滤、代码审计、沙箱验证，高风险 Skill 沙箱隔离监控
对应公司安全要求

skill 安全审查，skill 授权

缓释的原生 OpenClaw 风险
  • 智能体安全功能插件（skills）投毒风险
  • 系统安全安全漏洞风险
04
运行层面 · Runtime

强制经过 AI 安全护栏，操作可监控

  • AI 安全护栏能力保证（内容安全、防提示词注入、敏感信息泄露、token 限流、日志审计等）
  • 意图偏离、任务漂移异常识别
对应公司安全要求

内容安全，操作日志实时监控，大模型网关管理，降低 AI 操作失误风险

缓释的原生 OpenClaw 风险
  • 法律合规责任边界风险
  • 内容安全提示词注入攻击风险
  • 数据安全企业敏感数据、凭证与密钥明文泄露风险
在保证数据安全和权限可控、安全风险可接受的前提下，OpenClaw 可以打通内部应用 / 数据库，执行可管控的操作。
数据可控 权限收敛 全程审计
PART 02 · 应用场景

潜在应用场景

SCENARIO 01

用户团队进行敏感文档审查、敏感数据处理

EXAMPLE
  • 审计团队批量审查敏感文档、协助取证、审计底稿编写
  • 财务团队敏感数据分析、追溯账目、生成报表
SCENARIO 02

内部系统操作助手

EXAMPLE
  • 对话形式查询固定 SQL
  • 内部应用系统业务数据查询、报表生成
  • 系统辅助测试
SCENARIO 03

低权限 BAU 任务 7×24 处理

EXAMPLE
  • 低风险系统状态定时巡检，如检查接口可用性、定时任务执行状态、异常通知及预处理
PART 03 · 方案对比

“安全养虾”平台 vs 飞书 Aily

✓满足 / 企业可控 ◐部分满足 / 依赖第三方 ×不满足 / 不可控
主要对比维度 “安全养虾”平台 飞书 Aily 智能伙伴
环境隔离
✓每个 Openclaw 实例独立容器，多租户微隔离
✓局限于飞书环境内，飞书承诺沙箱隔离
敏感信息保护
✓数据在企业内，或被企业订阅的大模型管理
✓飞书环境内，飞书承诺数据保护
安全配置管理
✓企业可以集中管理配置
◐基于飞书控制，企业不可控，飞书承诺安全保障
提示词注入防护、内容合规、敏感数据保护
✓AI 护栏双向防护
◐平台内置基本内容审核，无企业专项需求
行为审计和日志
✓推理链路、工具调用等行为均可视
◐企业不可见，依赖飞书日志
供应链安全
✓安全审查、集中授权
✓未开放插件生态，安全控制不可见，飞书承诺安全保障
