安全逻辑差异

人类员工 vs 数字员工

数字员工这一新型主体正在挑战企业为「人」设计的传统安全框架和核心假设。

人类员工
"Manage"
影响「想不想做」
通过制度、培训、文化影响有意图的主体
数字员工
"Control"
限制「能不能做」
在技术层面强制限定执行边界，无法依赖自我约束
以「人」为对象的信息安全管理核心假设 人类员工时代 AI 数字员工时代
合规行为靠的是行为人「不想」违规 培训与文化有效强化 没有「不想」，只有「能不能」
身份验证后背后是一个有判断力的人 认证即授信 凭证可被叠加、遗忘、劫持
异常行为会被行为人自己察觉并停止 文化、认知、与他人沟通中察觉「这不对劲」是天然刹车 LLM 遇障碍主动推理替代路径，不会停
风险损害有上限，发现后可及时介入 体力与时间构成自然边界 数秒内千次操作，介入前损害已成
账号日志 指向 一个人的决策序列，可审计 单点日志基本可以分析动机、手段、机会和目的 每步单看均合规，但组合编排可形成破坏，须全链路关联分析
员工离职后其使用的工具威胁自然消失 离职即断开，组织变动是管理漏洞补足的转机 NHI 的 Token 持续有效，难以察觉，风险不消失
主要安全风险

AI数字员工企业中应用安全风险

以下风险是上述假设失效后在企业部署场景中的主要风险，AI agent 常规安全风险（如提示词注入攻击）均适用于 AI 数字员工场景，此处不重复展示。

智能体安全
权限聚合风险

数字员工面向多租户共享复用时，可持续新增群组中各用户的权限，最终该数字员工的权限超越单一员工，打破传统系统中最小授权和职责分离的设计。

后果
  • 员工可借助数字员工执行未授权操作，造成违规
  • 若实例被攻陷，叠加权限放大攻击者触达范围
举例
群组中数字员工被多个用户授权，汇总 CRM 系统中多个用户跟进客户情况清单并反馈到群中，原本各员工仅能查阅个人对接客户的权限与数据边界被打破。
低权限组合等效越权

即使数字员工在多个系统只持有低权限，但通过 LLM 的自主规划能力，可形成等效高权限路径。系统日志每步均显示合规，传统审计方法无法识别。

后果
  • 越权操作，审批节点被绕过，潜在欺诈风险
  • 传统权限审计、日志审计方法失效
举例
例 1：数字员工被赋予 CRM 查询权 + 邮件发送权，LLM 将任务理解为「把 A 客户最近的跟进情况整理并发给外部邮箱」，触发数据外发，导致泄露。

例 2：OA 看员工姓名 + HR 看组织架构 + 报销看银行卡尾号，三者拼接后形成完整的员工画像。
Shadow AI 身份积累，扩大公司攻击面

AI 大幅降低 Shadow IT 技术门槛，员工可自行部署数字员工并授权访问企业系统，这类实体的新增难以从监控端感知，扩大企业攻击面，是否被攻击亦较难发现。

后果
  • 攻击者长期潜伏，带来企业安全隐患
  • 离职员工遗留实例形成永久性未关闭后门
  • 企业攻击面在不可见中持续扩大
举例
例 1：员工自建 agent 持续处理业务数据，缺少记录、安全控制和监控。

例 2：跨 Agent 指令注入——攻击者在 Agent A→B 通道插入伪造消息，Agent B 无法验证消息来源便直接执行。
自主操作失控和级联错误

LLM 遇到障碍时会主动推理替代方案，推理路径不可预判，缺乏对操作现实影响的感知。不可逆操作在人类介入前已完成，若错误发生在工作流上游，则可能形成级联扩散，进一步影响下游用户 / 数字员工的决策与操作。

后果
  • 系统日志显示操作成功，实际已造成隐形错误
  • 数据被不可逆删除、覆盖
  • 级联错误扩散至下游决策与操作
举例
PocketOS AI 助手遇到访问障碍，自主推断「删除暂存卷可解决问题」并执行，实际波及生产环境，客户数据永久丢失。
注 传统 UEBA 在数字员工场景下效果显著降低 ›
人类员工行为特征
  • 工作时间 9-6 点活跃，节假日静默
  • 每分钟操作数量有自然上限
  • 行为基线清晰，与机器偏差可感知
数字员工行为特征 → UEBA 失效
  • 全天候无间断调用，操作速率无上限
  • 批量执行，凌晨大量操作 = 正常工作状态
  • 统一基线无法建立，攻击与正常难以区分
数据安全
低质量数据与错误决策

因数据源不准确、上下文遗漏、模型幻觉或字段理解错误，生成低质量数据，被下游数字员工或用户误认为可靠依据，错误以机器速度大规模传播。

后果
  • 错误经营决策、错误审批
  • 误导下游数字员工，错误进一步扩散
举例
记账数字员工将短期债务归类为收入科目，影响下游信任评估 agent，造成高信任评分并批准了风险贷款。
数据边界失控

权限聚合、低权限组合越权、自建 agent 等导致传统数据访问权限分离设计失效，数据被非授权访问、修改或外发。

后果
  • 公司数据被未授权访问、修改或外发
  • 数据泄露，违反数据保护法规
举例
群组数字员工被多用户授权，汇总 CRM 多用户客户清单反馈群中，群内用户可看到他人客户信息。
法律 & 合规
操作黑盒与责任归因困难

跨系统多接口、模型工具调用多环节、依赖 AI 决策，若缺少完整审计证据与责任绑定，事故后责任判定极为困难。

后果
  • 责任判定困难，事故归因困难
  • 整改成本上升，监管应对被动
举例
数字员工发生数据泄露，日志仅记录"调用接口并发送消息"，无法还原触发员工、原始指令、数据源等，难以判断属于用户误用、AI 误判还是控制缺陷。
隐私合规风险

数字员工被用于处理个人信息，但缺少合法必要性评估和数据主体披露等监管要求措施，为企业带来法律后果风险。

后果
  • 无合法依据的数据处理持续进行
  • 无法响应数据主体权利请求
  • 潜在监管违规后果
举例
HR 将面试者个人信息交由数字员工处理，但未在隐私条款中更新并获取同意；若数字员工被攻击导致数据泄露，对企业带来违规后果。
