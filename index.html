⚠ UNCERTAIN 其余所有情况

规则引擎 · 7 条规则明细
各规则分值累加，无上限（理论最高 91 分），再以权重 30% 贡献综合分。 移动端 Headers 不可用时 RULE-AUTH-001 自动跳过，上限降至 76 分。

威胁情报维度 · VirusTotal v3
  • 通过 /api/virustotal 代理调用，VT key 在服务端
  • 每次请求 4 秒 AbortController 超时
  • 每封邮件最多查询 6 个域名 / URL（VITE_INTEL_MAX_LOOKUPS 可调）
  • 重复域名去重，Promise.allSettled 并发查询
  • 恶意确认（≥ 2 引擎标记 malicious）→ +25 分
  • 可疑（1 引擎 malicious 或 ≥ 1 引擎 suspicious）→ +8 分
  • 情报分上限 60，免费账户 500 次 / 天
  • 失败（超时 / 限额 / key 缺）→ 静默忽略，不影响规则与 AI 维度
AI 分析维度 · Qwen / DashScope
  • EmailGuard 分析：qwen3.6-max-preview；EmailAssist 摘要：qwen3.6-flash；EmailAssist 回复：qwen3.6-plus
  • 所有 Qwen 调用统一走 /api/dashscope-attachment 代理
  • 输出 riskScore 0–100，乘以权重 0.50 后贡献综合评分
  • 输出 leaning（phish / uncertain / safe），用于裁决快捷路径
  • 所有调用均 enable_thinking: false，关掉推理链以保证 3-15s 延迟
  • 非流式（buffered）— 经 Function 代理后无法保留 SSE，UI 用 spinner 覆盖等待
06

EmailGuard · EmailAssist · AttachAnalyst 工作方式

同一个 Outlook 任务窗格内有 3 个 Tab，共享同一个 Office.js 邮件快照。 EmailGuard 负责钓鱼判断与处置； EmailAssist 负责摘要、翻译与回复建议； AttachAnalyst 负责附件深度分析（文本走 Qwen，图像走 Azure GPT-4.1-mini）。 三者完全独立，互不影响，可按需单独使用。

📌
共同入口 — 任务窗格 Tab 切换
用户在 Outlook 功能区点击「邮件分析 Lite」按钮 → 任务窗格打开 → 顶部并排显示 3 个 Tab：EmailGuard · EmailAssist · AttachAnalyst。 三者均读取 Office.context.mailbox.item 的同一份快照（发件人 / 收件人 / 主题 / 时间 / 附件数）， 正文与 Headers 在各自流程内按需异步读取，附件列表 / 字节由 AttachAnalyst 按需懒读。 任务窗格固定（Pin）后切换邮件时，三个 Tab 同步重置。
